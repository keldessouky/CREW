# CREW: Repository Skill Readiness

This repository contains the CREW process for evaluating repository support readiness, identifying required skills, and building redundancy across engineering teams.

## Included artifacts
- `SKILL.md` — workflow for generating repeatable repo skill readiness reports
- `template.md` — reusable report template for repo readiness assessments
- `atom-repo-skill-readiness-summary.md` — sample executive summary for the Atom repository

## How to use this skill
1. Open `SKILL.md` and follow the CREW workflow step-by-step.
2. For each repository you are evaluating, create a new report file using `template.md`.
   - Copy `template.md` into a new report such as `repo-name-skill-readiness.md`.
3. Fill in each section from the template:
   - Repo summary
   - Core support activities
   - Inferred tech stack
   - Critical skills
   - Priority ranking
   - Gap analysis
   - Redundancy plan
   - Learning and handoff actions
   - Risk reduction score
4. Keep the report team-focused, not person-focused.
   - Use roles and shared capabilities rather than individual names.
5. Save and share the completed report with the team.
   - Store it in the repository, a team workspace, or a shared knowledge base.
6. Apply the same process to every repo your team supports and review it regularly.

## Recommended output
- One finished readiness report per repository
- A consolidated list of skills by urgency and redundancy impact
- A plan for pairing, documentation, and backup ownership
- A repeatable process that your team can reuse each quarter
